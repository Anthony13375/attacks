### Задание 2

Проведите сканирование Metasploitable в режимах SYN, FIN, Xmas, UDP.

Запишите сеансы сканирования в Wireshark.

Ответьте на следующие вопросы:

* Чем отличаются эти режимы сканирования с точки зрения сетевого трафика?
* Как отвечает сервер?

*Приведите ответ в свободной форме.*

#### Ответ к заданию 2

SYN-сканирование. Сканер отправляет TCP-пакет с флагом SYN (синхронизация) на целевой порт. Это первый шаг стандартного TCP-рукопожатия.

Ответ сервера: 
* Порт открыт: Сервер отвечает пакетом с флагами SYN-ACK (подтверждение синхронизации). Получив его, сканер немедленно разрывает полуоткрытое соединение, отправляя пакет с флагом RST (сброс), чтобы не завершать рукопожатие полностью и не создавать запись в логах некоторых приложений
* Порт закрыт: Сервер отвечает пакетом с флагом
* Порт фильтруется: Если пакет SYN был отброшен брандмауэром, ответа не последует

![SYN](https://github.com/Anthony13375/attacks/blob/main/img/2.png)

FIN-сканирование.  Сканер отправляет TCP-пакет только с флагом FIN (окончание соединения). В норме такой пакет отправляется для завершения уже установленного соединения, поэтому его появление вне контекста выглядит аномально

Ответ сервера:
* Порт закрыт: Согласно RFC, закрытый порт должен ответить пакетом RST
* Порт открыт|фильтруется: Открытый порт игнорирует некорректный FIN-пакет и не отправляет никакого ответа. Отсутствие ответа может означать как открытый порт, так и фильтрацию, поэтому состояние помечается как open|filtered

![FIN](https://github.com/Anthony13375/attacks/blob/main/img/3.png)

Xmas-сканирование. Сканер отправляет TCP-пакет с несколькими флагами: FIN, PSH (push) и URG (urgent). Из-за множества "горящих" флагов пакет называется "рождественской елкой"

Ответ сервера:
* Порт закрыт: Ответ RST
* Порт открыт|фильтруется: Нет ответа

![XMAS](https://github.com/Anthony13375/attacks/blob/main/img/4.png)

UDP-сканирование. В отличие от предыдущих, это сканирование работает поверх протокола UDP. Сканер отправляет пустой UDP-пакет (или с полезной нагрузкой, специфичной для службы) на целевые порты

Ответ сервера:
* Порт открыт: Если на порту работает UDP-сервис, он может отправить ответный UDP-пакет
* Порт закрыт: Стек протоколов целевой системы отправляет ICMP-пакет с сообщением "порт недоступен"
* Порт фильтруется: Если UDP-пакет блокируется брандмауэром, может прийти другое ICMP-сообщение о недостижимости. Если нет ответа, порт также классифицируется как open|filtered, так как невозможно отличить фильтрацию от реально открытого, но молчаливого порта

![UDP](https://github.com/Anthony13375/attacks/blob/main/img/5.png)
